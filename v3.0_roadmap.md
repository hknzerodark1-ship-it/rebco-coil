# 𓂀 MERKLE FORTRESS v3.0 — THE 10ms HORIZON

## Current Baseline: 18.45ms (1,048,576 leaves)

## Optimization Targets:

### Phase 1: 15ms (18% improvement)
- [ ] Prefetch L2 cache lines (software prefetch)
- [ ] Reduce branch mispredictions in reduction loop
- [ ] Align slab arena to 64-byte cache lines
- [ ] Use `#[inline(always)]` on critical paths

### Phase 2: 12ms (33% improvement)  
- [ ] Implement SIMD-accelerated node hashing (ARM NEON)
- [ ] Batch leaf hashing with 4x vectorized Blake3
- [ ] Zero-copy leaf ingestion from memory-mapped files

### Phase 3: 10ms (45% improvement)
- [ ] Custom allocator for slab arena (bypass global allocator)
- [ ] Asynchronous promotion with tokio (reduce latency spikes)
- [ ] NUMA-aware thread pinning (big cores only for promotion)
- [ ] Use `std::arch::aarch64::__sevl` for power-aware waiting

## Implementation Timeline:
- Week 1: Phase 1 — Cache optimization
- Week 2: Phase 2 — SIMD batch hashing
- Week 3: Phase 3 — Custom allocator + async
- Week 4: Benchmark and release v3.0

## Stretch Goal: 8ms (56% improvement)
- [ ] Use secure enclave for leaf hashing (offload from main cores)
- [ ] FPGA accelerator for Blake3 (external, via USB-C)
